A finding from building Task #17

Reconstructed state is identical to uninterrupted execution for everything the
architecture actually asks to be identical -- belief confidence, plan/approval counts,
budget deltas, outcome success, ledger entries. It is not identical for internal record
IDs: Executor, Executive, Governor, and Memory & Ledger each mint a fresh
uuid4() inside the handler on every run rather than deriving it from the event, so

7/2/26, 11:05 PM                                                      Architecture Compliance Audit -- Update

action_id and friends differ between an original run and its replay. Not a defect --

those IDs were never part of the durable identity axiom -- but worth knowing before

anyone writes a test (or a downstream feature) that assumes replay reproduces IDs

byte-for-byte.
